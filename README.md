Hitting ball game: The Lab Formula (Advanced Edition)
A sophisticated and visually immersive version of the classic "Hitting ball game" logic game, set in a scientific laboratory environment. This project was built using Vanilla JavaScript with a focus on interactive UI and engaging user experience.

Project Overview
In this game, you must help the young scientist find the correct formula! The goal is to guess a secret 4-digit code generated by the system.

Bull: A correct digit in the exact correct position.

Pick: A correct digit that is in the wrong position.
The objective is to "turn on the lights" in the lab by cracking the code in as few turns as possible.

Tech Stack
JavaScript (ES6+): Core game logic, code generation algorithms, and guess validation.

HTML5: Semantic structure including registration, login, and the main game board.

CSS3: Custom lab styling using responsive backgrounds and smooth UI transitions.

Key Features
User Management System: Includes dedicated registration (register.html) and login (login.html) pages linked via JS logic.

Interactive Lab Interface: As shown in image_9469b9.jpg, the game features stylized input fields and a unique "Try Now" check button.

Game State Management: Tracks the number of attempts and provides real-time feedback on "Bulls" and "Picks."

Rich Visual Assets: Utilizes a dedicated gallery (the pictures folder in image_94699f.png) to create an authentic laboratory atmosphere.

Project Structure
Based on the file tree in image_94699f.png:

/html: Contains the index, login, registration, and main game pages.

/css: Styling files (games.css and ourGame.css) responsible for the unique aesthetic.

/js: The logic engine (main.js, login.js, register.js).

/pictures: All visual resources, including the laboratory background (background.jpg) and state-change images.

How to Run
Clone the repository to your local machine.

Navigate to the html folder.

Open login.html in any modern web browser to start the experience.

Developer Highlights
Algorithmic Logic: Efficient implementation of array comparisons to determine hits while preventing duplicates.

User Experience (UX): Immediate visual feedback provided for every user action.

Clean Architecture: Clear separation between the logical Backend in JS and the Frontend display in HTML/CSS.
